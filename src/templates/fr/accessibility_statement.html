Déclaration d’accessibilité

Aperçu

Le Web Almanac s’engage à assurer l’accessibilité numérique pour tous et toutes. Nous améliorons continuellement l’expérience des personnes et appliquons les normes d’accessibilité pertinentes.

Cette déclaration a été révisée pour la dernière fois le 2 Juin 2020.

Mesures pour assurer l’accessibilité de ce site

Le Web Almanac prend les mesures suivantes pour assurer l’accessibilité de ce site Web :

  • Inclure l’accessibilité dans le cadre du développement de ce site Web.
  • Vérifier l’accessibilité à l’aide des outils disponibles.
  • Être transparent sur les problèmes d’accessibilité sur notre dépôt Github.
  • Résoudre les problèmes d’accessibilité soulevés en temps opportun.
  • Publier et tenir à jour cette déclaration d’accessibilité.

Statut de conformité

Les directives pour l’accessibilité du contenu Web (WCAG) définissent les exigences pour les concepteurs et les développeurs afin d’améliorer l’accessibilité pour les personnes handicapées. Il définit trois niveaux de conformité : niveau A, niveau AA et niveau AAA. Le Web Almanac est partiellement conforme au niveau AAA des WCAG 2.1. Partiellement conforme signifie que certaines parties du contenu ne sont pas entièrement conformes à la norme d’accessibilité.

Nous nous considérons partiellement conformes parce que nous n’avons pas effectué un audit complet et vérifié de l’extérieur du site Web. Le site Web n’a pas non plus été examiné par des personnes utilisatrices de lecteurs d’écran, nous ne pouvons donc pas être sûrs de notre état actuel. Nous avons audité toutes les pages du site Web à l’aide d’un certain nombre d’outils, notamment l’outil d’accessibilité WAVE, l’inspecteur d’accessibilité Firefox et le plugin AX Chrome.

Le but de l’Almanac Web est de fournir des informations détaillées sur l’état du Web. Bien que nous cherchions à nous assurer que tout le contenu est facilement lisible, même par des personnes non-expertes, la nature du matériel peut signifier qu’il peut y avoir un niveau d’expertise et de compréhension technologique nécessaire pour comprendre certains chapitres et certaines mesures, qui peuvent ne pas répondre au niveau WCAG 2.1. des directives de niveau de lecture AAA.

Nous utilisons également Google Sheets pour certaines visualisations interactives, qui peuvent également ne pas être conformes aux normes d’accessibilité WCAG. Celles-ci sont chargées via des iframes, nous fournissons des images pour toutes nos illustrations et affichons uniquement des visualisations interactives lorsque le logiciel client le prend en charge. Si vous utilisez un navigateur Web et que vous ne souhaitez pas afficher les visualisations interactives, les images peuvent être appliquées en ajoutant un paramètre ?print pour tous les chapitres, consultez le chapitre Accessibilité en version imprimable pour un exemple de ce type. Nous pouvons chercher à rendre cette fonctionnalité plus facile à activer si nécessaire, mais toutes les visualisations ont des descriptions textuelles détaillées, nous ne pensons donc pas que cela soit nécessaire. Si vous pensez que c’est une erreur, n'hésitez pas à nous le dire.

Dans certains chapitres, nous incluons également d’autres contenus sur le site, non créés par nous-mêmes, y compris des vidéos YouTube, des liens PDF ou des liens vers d’autres articles, qui ne répondent pas aux critères stricts exigés par les WCAG 2.1 niveau AAA. Voir la note ci-dessous sur le contenu externe.

La version PDF du Web Almanac suit les meilleures pratiques d’accessibilité PDF et répond au standard PDF/UA-1.

Contenu externe

De nombreux chapitres renvoient à d’autres contenus (vidéos, PDF, articles) qui ne sont pas sous notre contrôle, et qui peuvent ne pas répondre aux niveaux élevés de conformité que nous avons atteints. Dans la mesure du possible, nous fournirons des liens vers des articles dans la même langue que la page consultée, mais ce n’est pas toujours possible.

Compatibilité avec les navigateurs et les technologies d’assistance

Le Web Almanac est conçu pour être compatible avec autant de navigateurs et de technologies d’assistance que possible. Le site Web a été construit de manière progressive et est pleinement fonctionnel même sans prendre en charge toutes les fonctionnalités de navigateur modernes que nous utilisons, mais vous pouvez remarquer une détérioration des visuels et un manque de fonctionnalités (visuels interactifs, nombre de commentaires, etc.) lorsque vous n’utilisez un navigateur moderne tel qu’une version récente de Chrome, Safari, Firefox ou Edge.

Pour le moment, nous n’avons pas terminé les tests du matériel ou des logiciels de technologie d’assistance, mais nous avons utilisé des outils, comme mentionné ci-dessus, pour tenter de résoudre les problèmes de ces appareils.

Spécifications techniques

Le site Web est construit sur du HTML sémantique bien structuré (y compris WAI-ARIA le cas échéant), amélioré avec CSS pour le style et JavaScript limité pour certains composants interactifs. Le site Web utilise des pratiques de développement modernes telles que les polices CSS Grid, Fetch et WOFF2, qui ne sont pas prises en charge dans les navigateurs plus anciens tels que IE 11. Le site Web est toujours entièrement fonctionnel dans ces navigateurs mais peut ne pas être aussi attrayant visuellement. Aucun plug-in ou bibliothèque supplémentaire ne devrait être nécessaire pour afficher le site Web.

Le Web Almanac PDF suit les mêmes normes d’accessibilité que le site Web et, en outre, est structurellement étiqueté pour faciliter l’accessibilité.

Approche d’évaluation

Auto-évaluation : le contenu a été évalué par l’équipe Web Almanac en utilisant sa propre expertise et un certain nombre d’outils dont l’outil d’accessibilité WAVE, le l’inspecteur d’accessibilité Firefox et le plugin AX Chrome. Le Web Almanac PDF a été testé avec une analyse d’accessibilité dans Acrobat Reader Pro DC.

Retour d’information

Vos commentaires sur l’accessibilité de ce site Web nous intéressent. La meilleure façon de nous joindre serait d’ouvrir un ticket sur Github ou de nous envoyer un courriel à team@httparchive.org. Vous souhaitez contribuer directement ? N’hésitez pas à proposer vos Pull Requests, elles seront les bienvenues. Le Web Almanac est un projet communautaire, géré par des bénévoles, et nous ne pouvons donc pas garantir un délai pour répondre aux commentaires, mais nous y répondrons dès que possible.
